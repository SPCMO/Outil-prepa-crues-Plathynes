Installation de l'outil

Avant le premier lancement, vérifiez que votre environnement Python est complet en exécutant le script de test fourni.

Étape 1 — Lancer le vérificateur d'environnement

1

Double-cliquez sur Test_pr_install.bat

Une fenêtre de commande s'ouvre et le script enchaîne automatiquement quatre vérifications :

  1. Version Python — vérifie que Python 3.9 à 3.13 est installé et accessible dans le PATH. Les versions antérieures sont bloquées ; les versions supérieures à 3.13 déclenchent un avertissement (non testées) mais ne bloquent pas.
  2. Packages requis — contrôle la présence et la version minimale de chaque dépendance :
    PackageVersion min.Rôle
    tkinterstdlibInterface graphique
    requests2.28Appels HTTP vers PHyC et BDImage
    lxml4.9Construction des enveloppes SOAP brutes
    zeep4.2Client SOAP (parsing WSDL, authentification)
    matplotlib3.6Graphiques (utilise l'API colormaps[] dispo depuis 3.5)
  3. Package optionnel — geopandas (≥ 0.13) : utilisé uniquement par le script tools/gen_bbox_bnbv.py pour générer les bounding boxes sur un nouveau territoire. Non nécessaire pour l'utilisation courante.
  4. Fichiers du projet — vérifie la présence de tous les fichiers indispensables : main.py, aide.html, les modules, config/bbox_bnbv.json et le dossier sorties/ (créé automatiquement s'il est absent).
2

Installation automatique des packages manquants

Si des packages requis sont absents, le script propose de les installer automatiquement via pip :

  Voulez-vous installer les packages REQUIS manquants maintenant ? [O/n] :

Répondez O (ou Entrée) pour lancer l'installation. Le script relance ensuite une vérification finale et affiche un bilan.

Si seuls des packages optionnels manquent (ex. geopandas), aucune installation n'est proposée — l'outil fonctionne sans.

3

Bilan affiché en fin de script

Le script se conclut par un bilan :

  • [OK] Tout est en ordre → l'outil peut être lancé via Lancer_OPALE.bat.
  • [!!] Des problèmes ont été détectés → consultez les lignes [ERR] pour identifier ce qui manque.
⚠️ Si pip ne parvient pas à télécharger les packages (erreur réseau / timeout)

Le réseau RIE filtre les accès à Internet et bloque PyPI (le dépôt de packages Python). Pour installer les dépendances, vous devez sortir du RIE le temps de l'installation :
  1. Branchez la clé 5G sur votre PC.
  2. Débranchez le câble réseau RJ45 du PC (ou désactivez la carte réseau filaire dans les paramètres Windows).
  3. Relancez Test_pr_install.bat et répondez O à la proposition d'installation — pip passera alors par la connexion 5G pour accéder à PyPI.
  4. Une fois les packages installés, rebranchez le câble RJ45 et débranchez la clé 5G.

⚠️ Ne laissez pas votre PC connecté simultanément au RIE et à Internet sans autorisation — respectez la politique de sécurité de votre entité.

Étape 4 — Lancer l'outil

4

Double-cliquez sur Lancer_OPALE.bat

L'interface graphique s'ouvre. Configurez ensuite votre station dans l'onglet Configuration (voir le Tutoriel utilisateur).

En cas d'erreur au démarrage, le bat affiche un message vous invitant à relancer Test_pr_install.bat pour diagnostiquer.

Guide d'utilisation — Outil préparation crues Plathynes

Ce guide vous accompagne pas à pas dans l'utilisation de l'outil, de la configuration initiale jusqu'à l'analyse comparative des épisodes de crue.

Avant de commencer

L'outil nécessite :

  • Un accès au réseau RIE (réseau interne SPC / SCV)
  • Un compte PHyC (identifiant + mot de passe)
  • Le code station hydrométrie de votre site (format Yxxxxxxxxx, 10 caractères — ex. Y027401001 pour Le Boulou). Le code site à 8 caractères est dérivé automatiquement des 7 premiers chiffres. Le préfixe (Y par défaut) et la longueur du suffixe sont configurables dans l'onglet ⚙ Paramétrage.
  • Le code BNBV du bassin versant pour les pluies/HU (format MOxxxxxx). Le préfixe (MO par défaut) est configurable dans l'onglet ⚙ Paramétrage.
La bounding box est précalculée — aucune saisie requise.
Le fichier config/bbox_bnbv.json contient déjà les coordonnées d'extraction (Lambert 93, marge de 5 km) pour l'ensemble des bassins versants du territoire SPCMO. Lorsque vous saisissez un code BNBV, la bounding box est chargée automatiquement. La marge de 5 km autour de chaque bassin garantit que les pixels de bordure BDImage sont inclus.
Modifier la marge de la bounding box : La marge par défaut est de 5 000 m (5 km). Pour l'ajuster, modifiez la constante MARGE_M dans tools/gen_bbox_bnbv.py (ex. MARGE_M = 3000 pour 3 km), puis relancez le script pour régénérer config/bbox_bnbv.json.

Étape 1 — Configuration initiale

1

Onglet Configuration

Section Station hydrologique : saisissez le code station hydrométrie complet sans le préfixe (Y par défaut, voir onglet ⚙ Paramétrage) — 9 chiffres (ex. 027401001 pour Le Boulou, soit Y027401001). Le code site à 8 caractères (Y0274010) est dérivé automatiquement des 7 premiers chiffres et utilisé pour les requêtes débit Q. Saisissez également le code BNBV sans le préfixe (MO par défaut), et le nom de la station. La bounding box est chargée automatiquement à partir du code BNBV.

Section PHyC : vérifiez l'URL WSDL, saisissez votre identifiant et mot de passe. Utilisez le bouton "Récup info PHyC" pour valider la connexion et récupérer le libellé.

Section Seuils de vigilance : choisissez d'abord la grandeur (Q en m³/s ou H en m). Saisissez les 6 seuils (Jaune, Orange, Rouge et leurs zones de transition ZT) dans l'unité correspondante. Pour Q, la requête PHyC utilise le code site (8 car.) ; pour H, elle utilise le code station complet (10 car.). Récupérez-les automatiquement via "Récupérer depuis PHyC", puis cliquez "Appliquer". Les deux jeux de seuils (Q et H) sont stockés indépendamment dans la configuration.

Cliquez "Enregistrer la configuration" pour sauvegarder.

Étape 2 — Définir les épisodes à traiter

2

Onglet Épisodes

Option A — Chargement depuis un catalogue : cliquez "Charger CSV épisodes..." pour importer un catalogue issu d'OCTAVE ou tout CSV avec colonnes date début / date fin. Les épisodes s'affichent dans le tableau.

Option B — Ajout manuel : saisissez une date de début et une date de fin (format JJ/MM/AAAA HH:MM), la durée se calcule automatiquement. Cliquez "+ Ajouter".

Sélectionnez les épisodes à extraire (Ctrl+clic ou "Tout sélectionner").

La colonne Vig. max. indique le niveau de vigilance maximal atteint pour chaque épisode parmi les 7 niveaux possibles : Vert (sous tous les seuils), ZT Jaune, Jaune, ZT Orange, Orange, ZT Rouge, Rouge. La ligne est colorée en conséquence (vert / tons jaunes / tons oranges / tons rouges).

Cliquez "↻ Rafraîchir" pour mettre à jour la colonne Vig. max. après une extraction ou un changement de seuils.

Filtrer les épisodes : le bandeau bleu sous le titre propose des filtres permanents (toujours visibles) :

  • Date début / Date fin — saisissez une date partielle (ex. 2020) pour ne garder que les épisodes dont la date de début ou fin contient cette chaîne.
  • Vig. max. — cases à cocher pour chacun des 7 niveaux ; décochez un ou plusieurs niveaux pour masquer les épisodes correspondants. Le filtre s'applique instantanément.
  • Cliquez "Réinitialiser" pour remettre tous les filtres à zéro.

Trier les colonnes : cliquez sur un en-tête de colonne (#, Date début, Date fin, Durée, Vig. max.) pour trier les épisodes dans l'ordre croissant, puis décroissant (▲ / ▼). Un nouveau clic sur le même en-tête inverse l'ordre.

Étape 3 — Lancer l'extraction

3

Onglet Extraction

Cochez les types de données à télécharger :

  • Antilope (case à cocher, pas de temps configurable) — pluies spatialisées temps-différé. Génère des fichiers .grd (ESRI ASCII Grid) dans {station}/Pluies/AntJ1-Ep_{date}/ et un CSV de pluie moyenne BV dans Pluies temps moy BV pour graph/AntJ1_BV-Ep_{date}.csv. Produit BDImage : antilope/j1 (60 mn) avant le 12/10/2020 ; antilope/france-td-60mn ou france-td-15mn après.
  • Panthère (case à cocher, pas de temps fixé à 1 heure — seul produit disponible sur BDImage : panthere/france) — pluies radar temps-différé. Génère des .grd dans {station}/Pluies/Pant-Ep_{date}/ et un CSV dans Pluies temps moy BV pour graph/Pant_BV-Ep_{date}.csv.
  • HU moyen BV (humidité des sols) — fichier CSV dans {station}/HU/, pas de temps : 1 heure (défaut) ou 1 jour à 06:00.
  • Débits / Hauteurs PHyC (Q ou H, pas de temps configurable) — fichier TXT dans {station}/Debits/.

Les préférences d'extraction (cases cochées, pas de temps) sont mémorisées automatiquement dans config.json à chaque lancement.

La barre de boutons propose à gauche "Lancer l'extraction" et "Arrêter", et à droite "Enregistrer la configuration" et "ℹ Aide" (identiques à ceux de l'onglet Configuration). Le journal affiche la progression en temps réel. À la fin, une fenêtre de synthèse récapitule les téléchargements épisode par épisode et indique la durée totale de l'extraction (ex. : Chargés en 12 min 34 s).

La liste Visualisation s'actualise dès qu'un fichier débit, Antilope ou Panthère est présent — il n'est pas nécessaire d'avoir extrait les 3 types de données pour visualiser un épisode.

Étape 4 — Visualiser un épisode

4

Onglet Visualisation

La liste de gauche affiche les épisodes disponibles, triés du plus récent au plus ancien. Chaque ligne débute par une pastille colorée (●) dont la couleur indique le niveau de vigilance maximal atteint. Un indicateur entre crochets précise les données disponibles : [Q] débit, [HU] humidité des sols, [Ant] pluie Antilope, [Pan] pluie Panthère. Un épisode peut apparaître dès qu'une seule de ces données est présente. Cliquez sur un épisode pour afficher :

  • Graphique haut — Hyétogramme inversé (barres vers le bas) + courbe HU en pointillé rouge.
    • Si Antilope est disponible : barres bleues (valeur ≤ seuil) et violettes (valeur > seuil), au premier plan (zorder élevé).
    • Si Panthère est disponible : barres orange semi-transparentes en arrière-plan, bordures en pointillés.
    • La légende : le cumul AJ1 est intégré directement dans le label "Antilope BV ≤ seuil mm" sur la même ligne. Le cumul Pan. est intégré dans le label "Panthère BV (mm)" avec un tiret de séparation.
    • Les annotations HU début et HU 6h indiquent les humidités clés.
  • Graphique bas — Courbe de débit Q avec zones de vigilance colorées. Les annotations Q début et Q max sont affichées avec horodatage.

Le graphique de débit affiche les lignes des 6 seuils de vigilance (ZT Jaune, Jaune, ZT Orange, Orange, ZT Rouge, Rouge) avec leur valeur en étiquette. Les seuils ZT sont tracés en pointillé/pastel, les seuils principaux en trait plein. Les étiquettes dont la valeur dépasse la plage visible du graphique sont masquées automatiquement pour ne pas déborder hors zone.

Le champ "Seuil hyétogramme *" (défaut 10 mm/pas) détermine la limite entre la coloration bleue et violette d'Antilope, et recalcule à la volée l'indicateur de fortes intensités dans l'encart de comparaison (voir ci-dessous).

Encart "Antilope vs Panthère" (bas-droite du graphique haut, visible si les deux produits sont présents) — 4 indicateurs comparatifs, chacun avec la valeur en mm et le pourcentage relatif par rapport à la valeur Antilope au même pas de temps :

  • Écart max 1h — pas de temps où l'écart |Pan.−Ant.| est le plus grand, avec horodatage.
  • Cumul 1h max Ant. — écart au pas de temps où Antilope est au pic, avec horodatage.
  • 3h consécutives max — fenêtre de 3 heures où Antilope est maximale, écart Pan.−Ant. sur cette fenêtre.
  • Forte int. (>X mm) — écart moyen de Pan. vs Ant. sur les seuls pas de temps où Antilope dépasse le seuil hyétogramme (recalculé si le seuil est modifié).

Les barres sont colorées en bleu si Antilope > Panthère, en orange si Panthère > Antilope. Leur longueur est proportionnelle au maximum observé sur l'ensemble des épisodes (référence globale propre à chaque indicateur). En pied d'encart : "Panthère sous/sur-estime X % des pas de temps (nb : N) soit Y % du cumul global".

Note sur l'astérisque (*) : ces hyétogrammes sont une reconstitution de la pluie moyenne de bassin versant calculée à partir des pluies spatialisées téléchargées (fichiers .grd). La moyenne est effectuée sur l'ensemble des pixels de la grille (y compris la marge de 5 km de la bounding box). Il ne s'agit pas d'une mesure directe mais d'une estimation spatiale — valable pour Antilope comme pour Panthère.

Les deux graphiques partagent la même échelle temporelle (x identique).

Étape 7 — Importer des crues dans Plathynes

7

Onglet Import Plathynes

Cet onglet permet de générer automatiquement les fichiers nécessaires à Plathynes (`.evt`, `.mrr`, `.mqo`) à partir des données extraites, et de les importer directement dans un projet Plathynes existant.

Section "Projet Plathynes cible"

Sélectionnez le fichier .prj du projet Plathynes dans lequel importer les épisodes. Une fois sélectionné, l'outil affiche automatiquement :

  • Le nom du bassin (Bassin: lu dans le .prj)
  • Le nombre et les noms des épisodes déjà présents dans le projet

Le bouton ↺ rafraîchit la liste des épisodes disponibles après modification du projet.

Section "Dossiers"

Deux sous-sections dans un même bandeau :

  • Sources : dossier contenant les fichiers Pluies extraits (ex. sorties/Sals/Pluies/) et dossier contenant les fichiers Débits extraits (ex. sorties/Sals/Debits/ et sorties/Sals/HU/). Ces champs sont pré-remplis depuis le dossier de sortie local si la configuration est renseignée.
  • Plathynes : répertoire d'installation de Plathynes (ex. C:\plathynes_v1.10.2). Utilisé pour trouver plathynes.bat et pour accéder à l'aide en ligne.

Section "Pas de temps de l'évènement (.evt)"

Quatre comboboxes pour configurer les pas de temps (valeurs possibles : 00:15, 00:30, 01:00) :

  • Forçage — pas de temps des données de pluie
  • Calcul — pas de temps interne du modèle
  • Sorties — pas de temps des résultats exportés
  • Bilans — pas de temps des bilans hydrologiques

Le bouton ℹ Aide Plathynes ouvre TOC_PLATHYNES_new.html dans le navigateur par défaut (si le répertoire Plathynes est renseigné). Ces valeurs sont mémorisées dans config.json.

Section "Crues extraites disponibles"

Liste des épisodes détectés dans les dossiers sources. Pour chaque épisode :

  • Case à cocher (décochée par défaut) — sélectionner les épisodes à importer
  • Couleur de fond — même palette que l'onglet Épisodes (vert / jaune / orange / rouge)
  • Grisé et barré — épisode dont la plage de dates chevauche un épisode déjà présent dans le .prj (non importable sans conflit)
  • Label "déjà importé" — épisode déjà présent dans le projet (également grisé/barré)

La molette souris fonctionne sur la liste lorsque le curseur est dessus. Un bandeau de filtres (Vert / ZT Jaune / Jaune / ZT Orange / Orange / ZT Rouge / Rouge + bouton ✕ Reset) permet de filtrer les épisodes ; les épisodes masqués par le filtre sont automatiquement décochés.

Import et ouverture

Cliquez "▶ Importer dans Plathynes" pour générer les fichiers de chaque épisode coché :

  1. .evt — fichier de définition de l'évènement avec les dates, les 4 pas de temps configurés, et la valeur d'humidité initiale HU (lue depuis la première ligne du fichier HU-Ep_*.csv). Si le fichier HU est absent pour un épisode, ce paramètre est omis mais un avertissement est affiché dans le journal en nommant l'épisode concerné.
  2. .mrr — liste des fichiers GRD de pluies observées
  3. .mqo — fichier de débits observés (format Plathynes)

Le projet .prj est mis à jour (nombre d'événements + ligne Evenement:), puis Plathynes est lancé automatiquement avec le projet via plathynes.bat et le chemin absolu du .prj.

Nommage des épisodes : le nom de l'évènement intègre la date et le niveau de vigilance maximal. Ex. : 10_04_2009_ZT_Jn (ZT Jaune), 29_11_2014_ZT_Or (ZT Orange), 15_10_2018_ZT_R (ZT Rouge).
Chevauchements : les épisodes dont les dates chevauchent des évènements déjà présents dans le .prj sont affichés en grisé/barré. Il est déconseillé de les importer, car Plathynes ne peut pas gérer deux évènements qui se superposent temporellement.

Étape 8 — Adapter les préfixes (optionnel)

8

Onglet ⚙ Paramétrage

Cet onglet (fond sombre, visuellement séparé) permet d'adapter l'outil à un autre territoire sans modifier le code source :

  • Préfixe du code station hydrométrie — lettre ou chaîne précédant les chiffres du code station (ex. Y pour les stations du SPCMO).
  • Nombre de chiffres — longueur du suffixe numérique (9 par défaut). Le code site est automatiquement déduit : préfixe + (n-2) premiers chiffres.
  • Préfixe du code BNBV — préfixe des bassins versants (ex. MO pour SPCMO).

Cliquez "Enregistrer le paramétrage". Les labels de l'onglet Configuration se mettent à jour immédiatement. Vérifiez ensuite que les codes station et BNBV saisis sont cohérents avec les nouveaux préfixes.

Conseils pratiques

  • Pour les stations sans seuils PHyC, saisissez les valeurs manuellement dans Configuration puis cliquez "Appliquer".
  • Si des .grd sont déjà présents, la pluie de bassin (PluieBV-Ep_*.csv) peut être régénérée sans re-télécharger en sélectionnant l'épisode dans Visualisation (auto-génération si manquant).
  • Les fichiers de débit PHyC contiennent des valeurs en m³/s arrondies au pas de temps grille configuré.
  • Utilisez "Tout désélectionner" puis sélectionnez uniquement les épisodes manquants pour une extraction partielle.

Bounding box pour un autre territoire

L'outil est pré-configuré pour le territoire du SPCMO. Les préfixes de codes station et BNBV se configurent directement depuis l'onglet ⚙ Paramétrage de l'application, sans modifier le code source. Cette page couvre uniquement la régénération de la bounding box des bassins versants pour un autre territoire.

Préfixes et longueurs de codes : pour adapter le préfixe du code station hydrométrie (ex. Y), le nombre de chiffres, ou le préfixe BNBV (ex. MO), utilisez l'onglet ⚙ Paramétrage de l'application — aucune modification de code nécessaire.

1. Régénérer la bounding box des bassins versants

Le fichier config/bbox_bnbv.json associe chaque code BNBV à ses coordonnées d'extraction (Lambert 93, marge 5 km). Pour un autre territoire :

  1. Procurez-vous le shapefile des BNBV de votre territoire (.shp + .dbf, projection Lambert 93 EPSG:2154, champ IDENTIF contenant les codes BNBV).
  2. Placez-le dans un dossier BNBV_MonSPC/ à la racine du projet.
  3. Éditez tools/gen_bbox_bnbv.py et modifiez les trois variables en tête de fichier :
SHP     = "../BNBV_MonSPC/BNBV_Bassins_MonSPC.shp"
DBF     = "../BNBV_MonSPC/BNBV_Bassins_MonSPC.dbf"
MARGE_M = 5000   # ← marge en mètres autour de chaque bassin (défaut : 5 km)
  1. Lancez : python tools/gen_bbox_bnbv.py — génère config/bbox_bnbv.json.
  2. Redémarrez l'outil : les nouveaux codes BNBV sont automatiquement reconnus.
Le script lit le shapefile en natif (sans geopandas) via les formats binaires .shp/.dbf. Aucune dépendance SIG n'est requise pour cette étape.

Fonctionnement interne du script

Pour chaque entité du shapefile :

  1. Lecture du .dbf → codes IDENTIF dans l'ordre des enregistrements.
  2. Lecture du .shp → bounding box géométrique brute (Xmin, Ymin, Xmax, Ymax) lue depuis les octets 4–35 de chaque record (doubles big-endian, format ESRI).
  3. Extension de MARGE_M mètres dans les 4 directions.
  4. Formatage : UL = "Xmin−M,Ymax+M" et LR = "Xmax+M,Ymin−M".
Pour modifier la marge sur un territoire existant, changez uniquement MARGE_M et relancez le script. Une marge plus petite réduit la zone téléchargée (plus rapide) ; une marge plus grande capture davantage de pixels de bordure.

Seuils de vigilance — structure JSON

Dans config.json, les seuils sont désormais stockés en deux jeux indépendants selon la grandeur :

{
  "station": {
    "code_station": "Y027401001",
    "code_site":    "Y0274010",
    ...
  },
  "seuils_q": {
    "zt_jaune":  120.0,
    "jaune":     150.0,
    "zt_orange": 250.0,
    "orange":    300.0,
    "zt_rouge":  450.0,
    "rouge":     500.0
  },
  "seuils_h": {
    "zt_jaune":  1.20,
    "jaune":     1.50,
    "zt_orange": 2.50,
    "orange":    3.00,
    "zt_rouge":  4.50,
    "rouge":     5.00
  }
}

Compatibilité ascendante : les anciens JSON avec une clé "seuils" sont lus comme seuils_q au démarrage. Les deux jeux peuvent coexister et sont configurés indépendamment via le sélecteur Q/H dans l'onglet Configuration.

Les ZT (Zones de Transition) apparaissent en pointillé/pastel sur le graphique ; les seuils principaux en trait plein/couleur prononcée. L'unité de l'axe Y s'adapte automatiquement (m³/s ou m) selon la grandeur de l'épisode affiché.

Historique des évolutions

Résumé chronologique des fonctionnalités développées lors des sessions de travail.

Fondations — Interface et extraction de base

Fonctionnalité Interface tkinter multi-onglets

Création de l'application avec 5 onglets (Configuration, Épisodes, Extraction, Visualisation, Analyse). Palette de couleurs par type de section (_make_section), ScrolledText pour le journal, barre de progression.

Fonctionnalité Authentification et extraction PHyC

Client SOAP PHyC via zeep avec transport RIE adapté (schémas XML W3C embarqués localement car inaccessibles depuis le réseau interne). Bypass zeep par enveloppe lxml brute pour publierObservationsHydroPasDeTemps.

Fonctionnalité Extraction pluies BDImage (.grd) et HU

Client BDImage pour téléchargement Antilope temps-différé (pluies spatialisées) et HU SIM par bassin versant. Écriture ESRI ASCII Grid.

Correction Correction des horodatages débits (heures rondes)

Les débits PHyC étaient retournés avec des timestamps décalés. Ajout de _snap_to_grid() pour arrondir au pas de temps configuré.

Visualisation

Fonctionnalité Onglet Visualisation avec graphique double

Deux sous-graphiques matplotlib (GridSpec 1:2) : hyétogramme inversé + HU en haut, courbe Q en bas. Sélection d'épisode depuis une liste avec indicateur de données disponibles.

UX Hyétogramme amélioré

Barres vers le bas (convention hydrologique), colorées en bleu/violet selon un seuil configurable (défaut 10 mm/pas). Auto-scale basé sur la pluie max (minimum 5 mm).

Fonctionnalité Annotations HU début / HU 6h

Points annotés sur la courbe HU : début d'épisode au-dessus, premier relevé à 6h00 en-dessous (ou 6h du lendemain si l'épisode commence après 6h).

Fonctionnalité Annotations Q début / Q max

Point de début et point max du débit annotés avec valeur en m³/s (exposant mathtext) et horodatage pour Q max.

Correction Synchronisation et finesse de l'axe X

Les deux graphiques partagent désormais exactement la même plage temporelle. Graduation adaptative : 3h (≤24h), 6h (≤72h), 12h (≤7j), 24h (au-delà). Correction du bug "01/01" causé par un xlim non défini après cla().

UX HU autoscale avec marge ×1.5

L'échelle de l'axe HU s'adapte à chaque épisode (min–max avec marge 50 %), au lieu d'un axe fixe 0–100 %.

Seuils de vigilance

Fonctionnalité Récupération automatique des seuils PHyC

Ajout de publierSeuilHydro dans le client PHyC. Parse les 6 niveaux (ZT Jaune/Jaune/ZT Orange/Orange/ZT Rouge/Rouge) filtrés sur NatureSeuil=22, actifs uniquement.

UX Zones de vigilance sur le graphique Q

Seuils ZT en pointillé/pastel, seuils principaux en trait plein/couleur prononcée. Zones de fond colorées. Échelle Y calée sur Q max uniquement (pas les seuils).

Correction Persistance des seuils dans la configuration

Correction du bug de rechargement : _refresh_config_ui() repopule maintenant les 6 champs seuils au démarrage. Les seuils sont inclus dans "Enregistrer la configuration".

Onglet Épisodes

Fonctionnalité Colonne Vig. max. avec colorisation des lignes

Pour chaque épisode, lecture du CSV Q, calcul du Q max, comparaison aux seuils. La ligne est colorée en jaune/orange/rouge selon le niveau atteint.

UX Bouton Rafraîchir + renommage de section

Section renommée "Chargement d'un catalogue de crues (OCTAVE par ex.)". Bouton "↻ Rafraîchir" pour recalculer la Vig. max. après extraction ou changement de seuils.

Pluie de bassin versant

Fonctionnalité Calcul automatique pluie BV depuis les .grd

Après extraction, la moyenne spatiale des .grd est calculée et écrite dans PluieBV-Ep_*.csv. Se régénère automatiquement si manquant lors de la sélection dans Visualisation.

Correction Fichiers .grd en entiers

force_integer=True activé pour les pluies : les valeurs float sont arrondies à l'entier pour compatibilité avec les logiciels de calage aval.

Onglet Analyse

Fonctionnalité Graphique bubble chart comparatif

Axe X = HU début, Y = Q max, taille = cumul pluie, couleur = intensité max. Lignes médianes et labels quadrants. Étiquettes date + Vig. max. avec placement greedy anti-superposition (8 directions).

Code station / code site et seuils H

Fonctionnalité Code station hydrométrie (10 car.) et dérivation automatique du code site

Le champ de saisie accepte désormais le code station complet : Y + 9 chiffres (ex. Y027401001 pour Le Boulou). Le code site à 8 caractères (Y0274010) est dérivé automatiquement par troncature des 7 premiers chiffres. Les deux codes sont persistés dans config.json sous station.code_station et station.code_site. Compatibilité ascendante avec les anciens JSON ne contenant que code_site.

Fonctionnalité Seuils de vigilance en hauteur H (m) en plus des seuils Q (m³/s)

La section "Seuils de vigilance" dispose d'un sélecteur Grandeur (Q m³/s / H m) qui bascule les unités affichées et le jeu de valeurs édité. Les seuils Q sont stockés dans seuils_q et les seuils H dans seuils_h — deux jeux indépendants dans config.json. La récupération PHyC utilise le code site pour Q et le code station complet pour H. Le graphique de débit adapte son libellé d'axe Y et la colonne Vig. max. lit le bon jeu de seuils selon la grandeur extraite.

Onglet Analyse

Correction Couleur des pastilles — compatibilité matplotlib 3.9+

matplotlib.cm.get_cmap() a été supprimé dans matplotlib 3.9. L'appel levait une exception silencieuse qui faisait tomber le code dans le fallback bleu uniforme (toutes les pastilles de la même couleur). Corrigé en utilisant matplotlib.colormaps["YlOrRd"] (API actuelle). La colorbar d'intensité pluie est également corrigée. Profité pour étendre la lecture des fichiers de métriques aux épisodes en hauteur (H-Ep_*.txt) en plus des débits (Q-Ep_*.txt).

Extraction — robustesse

Fonctionnalité Synthèse post-extraction (popup)

Fenêtre modale récapitulant par épisode : données téléchargées, lacunes détectées, erreurs.

Fonctionnalité Relance automatique en cas d'erreur

Si un épisode échoue, une relance immédiate est effectuée. Le résultat (succès ou échec définitif) apparaît dans le journal et la popup de synthèse.

UX Détection et signalement des lacunes

Vérification de la continuité temporelle pour Q, HU et pluies. Les lacunes sont listées dans le journal (max 5) et la popup de synthèse.

Filtres, tri et niveaux de vigilance

Fonctionnalité Bandeau de filtres permanent dans l'onglet Épisodes

Un bandeau bleu toujours visible sous le titre de la liste propose deux filtres indépendants : champ texte sur la date de début et la date de fin (filtrage instantané par correspondance de chaîne), et 7 cases à cocher pour le niveau de vigilance Vig. max. (Vert, ZT Jaune, Jaune, ZT Orange, Orange, ZT Rouge, Rouge). Un bouton "Réinitialiser" remet tous les filtres à zéro. Les filtres s'appliquent en temps réel via trace_add("write", …) sur les BooleanVar et StringVar.

Fonctionnalité Tri des colonnes dans l'onglet Épisodes

Cliquer sur un en-tête de colonne (#, Date début, Date fin, Durée, Vig. max.) trie la liste dans l'ordre croissant ; un second clic inverse l'ordre (▲ / ▼). Le tri de la colonne Vig. max. utilise un dictionnaire d'ordre (_VIG_ORDER) : Vert=0, ZT Jaune=1, Jaune=2, ZT Orange=3, Orange=4, ZT Rouge=5, Rouge=6.

Fonctionnalité 6 niveaux de vigilance ZT + niveau Vert

Les calculs de Vig. max. prennent désormais en compte les 6 seuils configurés (zt_jaune, jaune, zt_orange, orange, zt_rouge, rouge) vérifiés dans l'ordre décroissant. Les épisodes dont le débit reste sous tous les seuils reçoivent le niveau Vert (auparavant non distingué). Ce niveau apparaît dans les filtres, la colonne Vig. max. et les étiquettes du graphique Analyse.

UX Colorisation des épisodes Vert dans les onglets Épisodes et Visualisation

Les lignes "Vert" du Treeview (onglet Épisodes) sont colorées en vert clair (#D5F5E3 / #1D6A39) via le tag vig_vert. La même palette est appliquée aux items de la Listbox de l'onglet Visualisation.

Fonctionnalité Filtre Vig. max. dans l'onglet Analyse

Cases à cocher pour les 7 niveaux de vigilance, centrées dans le bandeau de contrôle entre le titre et le bouton "Calculer / Rafraîchir". La sélection filtre les épisodes inclus dans le bubble chart ; la mise à jour est immédiate à chaque clic.

Correction Duplication de la colorbar dans l'onglet Analyse

Chaque appel à _refresh_analyse() ajoutait une nouvelle colorbar sans supprimer l'ancienne, réduisant progressivement la zone graphique. Corrigé en remplaçant ax.cla() par fig.clear() + fig.add_subplot(111), qui efface l'intégralité de la figure (axes et objets figure-level comme les colorbars) avant chaque tracé.

Fonctionnalité Logo et icône de l'outil

Création d'un logo vectoriel (logo_OPALE.svg) représentant un hydrogramme de crue méditerranéenne : long plateau de débit de base, montée quasi-verticale, décrue exponentielle lente, point rouge au pic, trois seuils de vigilance en pointillés colorés, goutte d'eau et silhouette de montagne sur fond bleu nuit arrondi. Au démarrage, self.iconbitmap("logo_OPALE.ico") remplace l'icône par défaut de la fenêtre tkinter. Dans l'onglet Configuration, le logo (28×28 px, chargé via Pillow/ImageTk) s'affiche dans le bandeau bas à droite du lien contact PIOT Charles-Eddy — SPCMO.

Onglet Visualisation — tri et pastilles

UX Tri par date décroissante dans la liste des épisodes

Les épisodes sont triés du plus récent au plus ancien. Le tri est effectué sur la date parsée depuis le nom de fichier (DD_MM_YYYY → datetime), et non sur la chaîne alphabétique, afin d'obtenir un ordre chronologique correct quel que soit le jour du mois.

UX Pastilles de vigilance dans la liste Visualisation

Chaque item de la Listbox est préfixé d'un ● et coloré (fond + texte) selon le niveau de vigilance maximal atteint, calculé à la volée par lecture du fichier Q extrait (_vig_from_file()). Les couleurs sont cohérentes avec les tags du Treeview (onglet Épisodes) et la classe _VIG_ITEM_COLORS.

Onglet Paramétrage

Fonctionnalité Onglet Paramétrage territorial (fond sombre, visuellement distinct)

Nouvel onglet en dernière position permettant de configurer sans modifier le code source : le préfixe du code station hydrométrie (ex. Y), le nombre de chiffres du suffixe numérique (9 par défaut, le code site = préfixe + n-2 premiers chiffres), et le préfixe du code BNBV (ex. MO). Les valeurs sont persistées dans config.json sous la clé "parametrage" et lues via les propriétés _pfx_station, _nb_digits, _pfx_bnbv. À la sauvegarde, les labels de l'onglet Configuration se mettent à jour immédiatement.

Onglet Import Plathynes

Fonctionnalité Nouvel onglet Import Plathynes — génération automatique des fichiers évènement

Ajout du module modules/plathynes_importer.py et de l'onglet dédié dans l'interface. À partir des données extraites (pluies .grd, débits Q-Ep_*.txt, humidité HU-Ep_*.csv), l'outil génère automatiquement pour chaque épisode : le fichier .evt (définition de l'évènement), le fichier .mrr (liste des .grd de pluie) et le fichier .mqo (débits observés au format Plathynes). Le projet .prj est mis à jour puis Plathynes est lancé avec le chemin absolu du projet.

Fonctionnalité Génération du fichier .evt avec HU initial

Le fichier .evt contient les dates de début/fin (lues depuis le fichier Q), les 4 pas de temps configurables (forçage, calcul, sorties, bilans), et si disponible une section # Parameters avec la valeur d'humidité initiale HU : Parametre evenementiel: U <val> HU. La valeur est lue depuis la première ligne de données du fichier HU-Ep_*.csv. Si le fichier HU est absent, le paramètre est omis et un avertissement est affiché dans le journal en nommant l'épisode concerné.

Fonctionnalité Pas de temps configurables via comboboxes

Les quatre pas de temps (forçage, calcul, sorties, bilans) sont configurables indépendamment via des comboboxes (valeurs : 00:15, 00:30, 01:00). Les valeurs sont mémorisées dans config.json sous plathynes.pdt_forcage, pdt_calcul, pdt_sorties, pdt_bilans. Le bouton ℹ Aide Plathynes ouvre TOC_PLATHYNES_new.html dans le navigateur système (si le répertoire Plathynes est renseigné).

Fonctionnalité Détection des chevauchements de dates avec épisodes existants

Avant l'affichage de la liste, l'outil lit les dates de début/fin de tous les évènements déjà présents dans le projet .prj (via lire_plages_evenements()). Les épisodes dont la plage de dates chevauche un évènement existant sont affichés en grisé et barré (police overstrike tkinter) et décochés d'office. Les épisodes déjà importés (nom présent dans le .prj) sont également grisés/barrés.

UX Comportement de la liste des épisodes

Toutes les épisodes sont décochés par défaut. La molette souris fonctionne sur la liste lorsque le curseur la survole (bind_all/unbind_all sur <Enter>/<Leave> du canvas). L'application d'un filtre décoche automatiquement les épisodes masqués. La liste se rafraîchit automatiquement après chaque sélection de dossier via le bouton "Parcourir".

Fonctionnalité Dossier d'installation Plathynes et ouverture avec chemin absolu

Un champ "Plathynes :" dans la section Dossiers permet de renseigner le répertoire d'installation de Plathynes (pré-rempli à C:\plathynes_v1.10.2). L'outil y cherche plathynes.bat directement, avec un fallback sur les dossiers parent/grand-parent. Le projet est ouvert avec son chemin absolu (os.path.abspath()) car plathynes.bat fait un cd bin avant de lancer l'exécutable, rendant les chemins relatifs invalides.

Correction Renommage "Dossier de sortie" → "Dossier de sortie local" dans l'onglet Configuration

Le champ était ambigu (l'utilisateur pouvait croire que c'est le dossier cible Plathynes). Renommé en "Dossier de sortie local" avec un bouton ℹ expliquant son rôle : espace de travail local pour les fichiers extraits (pluies, débits, HU), distinct des dossiers d'import Plathynes.

Graphique débit — seuils et mise en page

Fonctionnalité Légende taille des bulles dans l'onglet Analyse

Une légende "Cumul pluie *" est affichée sous la barre d'intensité des pluies, avec son bord gauche aligné sur le bord gauche de la colorbar. L'alignement est garanti par la lecture de cb.ax.get_window_extent() (coordonnées pixels réelles) après un premier draw(), converti en coordonnées figure. Elle présente 3 bulles grises représentant 25 %, 50 % et 100 % du cumul maximum des épisodes chargés. La note explicative est positionnée immédiatement sous la légende via leg.get_window_extent().y0 après un deuxième draw(), avec un léger espace. Trois appels draw() au total.

Fonctionnalité Labels des 6 seuils de vigilance sur le graphique de débit

Les seuils ZT Jaune, ZT Orange et ZT Rouge affichent désormais leur étiquette (nom + valeur) au même titre que Jaune, Orange et Rouge. Les seuils ZT sont tracés en pointillé/pastel, les seuils principaux en trait plein. L'unité est dynamique (m³/s ou m selon la grandeur). Un label n'est affiché que si sa valeur est inférieure ou égale à Q_max × 1.20 — les seuils qui dépassent la plage visible du graphique restent tracés (ligne et zone de fond) mais sans texte débordant.

UX Optimisation des marges du graphique Visualisation

Les marges autour des deux sous-graphiques (hyétogramme/HU et débit) sont définies explicitement dans GridSpec (left=0.07, right=0.93, top=0.97, bottom=0.06, hspace=0.28) pour maximiser la zone utile. L'appel tight_layout() — incompatible avec les axes twinx et source d'un UserWarning dans le terminal — est conservé mais silencé via warnings.catch_warnings().

Correction Suppression de toutes les constantes codées en dur

Les valeurs "Y", "MO" et 9 étaient codées en dur à 9 endroits dans main.py (_save_config, _refresh_config_ui, _recup_info_phyc, _recup_seuils_phyc, _run_extraction, _on_bnbv_change et les labels UI). Elles sont désormais toutes remplacées par les propriétés dynamiques.
